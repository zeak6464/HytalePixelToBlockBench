# EQ (Audio Equalization)

Learn how to configure audio equalization to filter and modify frequency ranges for different environments.

## Overview

EQ (Equalization) settings control frequency filtering for audio. EQs are used to simulate different listening conditions, like underwater audio or distance filtering.

## Location
`Server/Audio/EQ/`

## Example from Game Files

### Underwater EQ

From `Server/Audio/EQ/EQ_Underwater.json`:

```1:12:Server/Audio/EQ/EQ_Underwater.json
{
  "LowGain": 0,
  "LowCutOff": 300,
  "LowMidGain": -17.19,
  "LowMidCenter": 1000,
  "LowMidWidth": 1,
  "HighMidGain": -17.9,
  "HighMidCenter": 1500,
  "HighMidWidth": 1,
  "HighGain": -17.9,
  "HighCutOff": 4000
}
```

This shows an EQ configuration for underwater audio with frequency filtering to simulate underwater sound.

## Basic EQ Structure

Create `Server/Audio/EQ/EQ_MyCustom.json`:

```json
{
  "LowGain": 0,
  "LowCutOff": 200,
  "LowMidGain": 0,
  "LowMidCenter": 500,
  "LowMidWidth": 1,
  "HighMidGain": 0,
  "HighMidCenter": 3000,
  "HighMidWidth": 1,
  "HighGain": 0,
  "HighCutOff": 6000
}
```

## EQ Properties

### Low Frequency Band

```json
{
  "LowGain": 0,
  "LowCutOff": 200
}
```

- **`LowGain`** - Gain/attenuation for low frequencies (dB)
- **`LowCutOff`** - Low-pass cutoff frequency (Hz)

### Low-Mid Frequency Band

```json
{
  "LowMidGain": -17.19,
  "LowMidCenter": 1000,
  "LowMidWidth": 1
}
```

- **`LowMidGain`** - Gain for low-mid frequencies (dB)
- **`LowMidCenter`** - Center frequency (Hz)
- **`LowMidWidth`** - Bandwidth (Q factor)

### High-Mid Frequency Band

```json
{
  "HighMidGain": -17.9,
  "HighMidCenter": 1500,
  "HighMidWidth": 1
}
```

- **`HighMidGain`** - Gain for high-mid frequencies (dB)
- **`HighMidCenter`** - Center frequency (Hz)
- **`HighMidWidth`** - Bandwidth (Q factor)

### High Frequency Band

```json
{
  "HighGain": 0,
  "HighCutOff": 6000
}
```

- **`HighGain`** - Gain/attenuation for high frequencies (dB)
- **`HighCutOff`** - High-pass cutoff frequency (Hz)

## Common EQ Presets

### Default

`Server/Audio/EQ/EQ_Default.json`:

```json
{
  "LowGain": 0,
  "LowCutOff": 200,
  "LowMidGain": 0,
  "LowMidCenter": 500,
  "LowMidWidth": 1,
  "HighMidGain": 0,
  "HighMidCenter": 3000,
  "HighMidWidth": 1,
  "HighGain": 0,
  "HighCutOff": 6000

```

No filtering, flat response.

```json
  "LowCutOff": 300,
  "LowMidGain": -17.19,
  "LowMidCenter": 1000,
  "LowMidWidth": 1,
  "HighMidGain": -17.9,
  "HighMidCenter": 1500,
  "HighMidWidth": 1,
  "HighGain": -17.9,
  "HighCutOff": 4000
}
```

Muffled, low-pass filtered sound for underwater.

## Using EQ in AmbienceFX

EQ can be applied via AmbienceFX conditions:

```json
{
  "Conditions": {
    "EnvironmentIds": ["Underwater"]
  },
  "EQ": "EQ_Underwater",
  "AmbientBed": {
    "Track": "Sounds/Environments/Underwater.ogg"
  }
}
```

## Tips for Creating EQ

1. **Underwater effect** - Reduce high frequencies (`HighGain` negative)
2. **Distance effect** - Reduce high-mid and high frequencies
3. **Low-pass filtering** - Lower `HighCutOff` for muffled sound
4. **Test with audio** - EQ effects need real audio to evaluate
5. **Combine with Reverb** - EQ + Reverb = realistic acoustic spaces
